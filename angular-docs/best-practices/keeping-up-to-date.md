# Keeping Your Angular Projects Up-to-Date
> Source: https://angular.dev/update

## Overview

Angular continuously improves while maintaining stability and straightforward update processes. The framework emphasizes both innovation and reliable upgrade paths for developers.

## Getting Notified of New Releases

Stay informed about Angular updates through:
- Following [@angular](https://x.com/angular) on X (formerly Twitter)
- Subscribing to the [Angular blog](https://blog.angular.dev)

## Learning About New Features

The Angular blog publishes [release announcements](https://blog.angular.dev/) highlighting major changes and new capabilities. For comprehensive details, consult the [Angular change log](https://github.com/angular/angular/blob/main/CHANGELOG.md) organized by version.

## Checking Your Angular Version

Use the command:
```bash
ng version
```
Run this from your project directory to identify your current Angular version.

## Finding the Current Version

The latest stable Angular version appears on [npm](https://www.npmjs.com/package/@angular/core) under "Version." You can also discover available updates using:
```bash
ng update
```
This command without arguments lists available updates for your application.

## Updating Your Environment and Apps

The interactive [Angular Update Guide](https://update.angular.dev) provides customized instructions based on your current and target versions. It includes:
- Basic and advanced update paths
- Troubleshooting information
- Recommended manual changes

For simple updates, the CLI command suffices. For detailed information about expected changes based on version numbers and supported paths, review the Angular versioning and releases documentation.
